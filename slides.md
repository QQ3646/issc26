# Анонимные функции
## В Scala

1. η-расширение
```scala
def increment(x: Int): Int = x + 1
val list = List(1, 2, 3)

list.map(increment)
```
Метод <code>increment</code> превращается в лямбду <code>{ x => increment(x) }</code>

# Анонимные функции
## В Scala

1. η-расширение
2. Параметры по имени (By-name parameters)
```scala
def log(message: => String): Unit = {
  if (debug) println(message)
}

log("User " + id + " logged in")
```

<code>"User " + id + " logged in"</code> превращается в лямбду <code>\{ () => "User " + id + ... \}</code> 
Преобразование в строку и конкатенация будут отложены до вызова функции <code>message</code>

# Анонимные функции
## В Scala

1. η-расширение
2. Параметры по имени (By-name parameters)
3. Частично примененные функции (Partially Applied Functions)

```scala
def sum(a: Int, b: Int): Int = a + b
val addFive = sum(5, _: Int)
```

<code>sum(5, _: Int)</code> превращается в лямбду <code>\{ x => sum(5, x) \}</code>

# Постановка задачи
## Проблема

Каждая из анонимных функций в условиях *JVM* (*Java Virtual Machine*) --- это полноценный объект, который требует как аллокации, так и сборки мусора. 
Из-за чего возникают издержки по памяти и времени.
